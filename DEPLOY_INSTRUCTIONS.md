## PART 1 — VPS Server Setup (via Hostinger Browser Terminal)

### Step 1: Open Hostinger Terminal
1. Navigate to https://hpanel.hostinger.com
2. Go to **VPS** section and find the KVM1 server at IP `187.77.175.220`
3. Click **"Connect to server"** or **"Browser terminal"** / **"VNC"** to open a web-based SSH session
4. Log in as `root`

### Step 2: Run this entire block in the terminal

```bash
# Create deploy user and give Docker access
adduser --disabled-password --gecos "" deploy
usermod -aG docker deploy

# Set up SSH directory for deploy user
mkdir -p /home/deploy/.ssh
chmod 700 /home/deploy/.ssh

# Add the GitHub Actions public key
echo "ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAICjXxksk5iHWJVy24wfTVhkF0A5pgG43RaXfP7qrK29h github-actions-deploy" >> /home/deploy/.ssh/authorized_keys
chmod 600 /home/deploy/.ssh/authorized_keys
chown -R deploy:deploy /home/deploy/.ssh

echo "✓ Deploy user created and SSH key added"
```

### Step 3: Install Docker (if not already installed)

```bash
# Check if Docker is installed
docker --version || (
  echo "Installing Docker..." &&
  curl -fsSL https://get.docker.com | sh &&
  systemctl enable docker &&
  systemctl start docker
)

# Confirm deploy user has docker access
groups deploy
echo "✓ Docker ready"
```

### Step 4: Clone the repo and create the .env file

```bash
# Switch to deploy user and clone
su - deploy -c "git clone https://github.com/CalvinMagezi/islas.git /home/deploy/islas"

# Create the .env file (fill in the values below)
cat > /home/deploy/islas/.env << 'EOF'

OPENROUTER_API_KEY=sk-or-replace-with-your-key

# Auto-generated by setup.sh — leave blank

# Generate these with: openssl rand -hex 32

WORKER_SECRET=replace-with-hex-secret

# VAPID keys — generate at: https://web-push-codelab.glitch.me

# Optional

EOF

echo "✓ .env template created at /home/deploy/islas/.env"
echo "⚠  Edit /home/deploy/islas/.env with real values before running setup.sh"
```

### Step 5: Generate secrets inline

```bash
echo "=== Generated secrets — copy these into your .env ==="
echo "ISLAS_API_KEY=$(openssl rand -hex 32)"
echo "WORKER_SECRET=$(openssl rand -hex 32)"
```

### Step 6: Once .env is filled in, run first-time setup

```bash
su - deploy
cd /home/deploy/islas
nano .env   # Fill in DOMAIN, ACCESS_PASSPHRASE, OPENROUTER_API_KEY, ISLAS_API_KEY, WORKER_SECRET, VAPID keys

# Then run setup
chmod +x docker/setup.sh
./docker/setup.sh
```

## PART 2 — GitHub Secrets Setup (via GitHub UI)

### Step 1: Navigate to repo secrets
1. Go to https://github.com/CalvinMagezi/islas/settings/secrets/actions
2. You may be prompted to log in — do so

### Step 2: Add these 4 secrets (click "New repository secret" for each)

| Secret Name | Value |
|-------------|-------|
| `VPS_HOST` | `187.77.175.220` |
| `VPS_USER` | `deploy` |
| `VPS_DEPLOY_PATH` | `/home/deploy/islas` |
| `VPS_SSH_KEY` | *(see below)* |

For `VPS_SSH_KEY` — the user must run this in their local terminal and paste the output:
```bash
cat ~/.ssh/islas_deploy
```
Copy the **entire output** including `-----BEGIN OPENSSH PRIVATE KEY-----` and `-----END OPENSSH PRIVATE KEY-----`

### Step 3: Verify workflows exist
1. Go to https://github.com/CalvinMagezi/islas/actions
2. You should see two workflows: **"CI"** and **"Deploy to VPS"**
3. If they show as disabled, click **"I understand my workflows, go ahead and enable them"**

### Step 4: Trigger first deploy
1. On the Actions page, click **"Deploy to VPS"**
2. Click **"Run workflow"** → **"Run workflow"** (green button)
3. Watch the logs — it should SSH into the VPS and run `docker compose up -d --build`
